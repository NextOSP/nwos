Contributing to NextOSP
=======================

Please open pull requests against the appropriate NextOSP branch and include enough context for reviewers
to reproduce and validate the change.

TL;DR

* If you make a pull request, do not create a duplicate issue. Use the PR description for that
* Issues are handled with a much lower priority than pull requests
* Please search for duplicates before reporting an issue
* Pull requests must be made against the correct version
* Stable series should only receive low-risk fixes
